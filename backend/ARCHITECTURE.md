## 二、服务职责与通信

| 服务     | 端口  | 职责                    | 通信方式       |
|---------|------|-----------------------|--------------|
| API网关  | 8080 | 统一入口、认证、限流、路由    | HTTP/gRPC    |
| 用户服务  | 5001 | 注册登录、用户管理、JWT签发   | gRPC内部通信   |
| 订单服务  | 5002 | 订单CRUD、状态管理、订单导出  | gRPC内部通信   |
| 行情服务  | 5003 | 实时行情、K线生成、WebSocket | gRPC + WS   |
| 账户服务  | 5004 | 资金管理、持仓管理、TCC事务  | gRPC内部通信   |
| 撮合服务  | 5005 | 订单撮合、成交生成、价格确定   | gRPC内部通信   |
| 对账服务  | 5006 | 资金对账、持仓对账、成交对账   | gRPC内部通信   |

## 三、极端情况处理清单

### 3.1 资金安全类

| 场景               | 处理方案                                    | 技术手段                    |
|------------------|-------------------------------------------|---------------------------|
| **并发超扣资金**      | 行级锁 + 乐观锁 + TCC补偿                      | WHERE cash >= amount + 事务回滚 |
| **重复扣款**        | 事务ID去重 + 幂等校验                           | Redis SetNX + 状态检查         |
| **冻结资金丢失**      | TCC Cancel补偿                              | Seata自动恢复                  |
| **持仓超卖**        | 持仓校验 + 事务 + 乐观锁                        | available_quantity校验         |
| **资金为负**        | 余额预校验 + 行级锁 + 告警                      | 双重保护 + 自动告警              |
| **对账不平**        | 定时对账 + 自动修复 + 人工干预                   | 差异检测 + 修复脚本              |

### 3.2 订单处理类

| 场景               | 处理方案                                    | 技术手段                    |
|------------------|-------------------------------------------|---------------------------|
| **重复下单**        | 幂等校验 + 订单去重                           | client_order_id去重          |
| **订单重复提交**     | 前端防抖 + 后端幂等                           | Redis缓存 + 状态检查          |
| **撤单时机错误**     | 状态校验 + CAS更新                            | 状态机保护                   |
| **部分成交后撤单**    | 只撤未成交部分 + 资金解冻                      | remaining_quantity计算       |
| **废单处理**        | 状态标记 + 资金释放 + 原因记录                  | reject_reason字段            |
| **订单超时**        | 定时扫描 + 自动取消 + 资金释放                  | cron定时任务                 |

### 3.3 系统异常类

| 场景               | 处理方案                                    | 技术手段                    |
|------------------|-------------------------------------------|---------------------------|
| **网络断连**        | WebSocket自动重连 + 心跳检测 + 断线提示            | ws重连机制 + ping/pong       |
| **Kafka不可用**    | 本地队列缓冲 + 降级处理 + 恢复重放                   | 本地队列 + 事件重放           |
| **数据库主从切换**   | 读写分离 + 自动重试 + 故障转移                     | 连接池管理 + 重试机制         |
| **Redis缓存失效**   | 回源数据库 + 熔断保护 + 缓存预热                    | Breaker模式 + 缓存穿透保护     |
| **撮合引擎崩溃**    | 订单状态恢复 + 内存重建 + 断点续传                  | 从MySQL恢复订单簿             |
| **TCC事务超时**    | Seata自动Cancel + 事务补偿                      | TC协调器自动处理              |
| **服务重启**        | 数据持久化 + 状态恢复 + 无缝重启                   | graceful shutdown          |

### 3.4 交易规则类

| 场景               | 处理方案                                    | 技术手段                    |
|------------------|-------------------------------------------|---------------------------|
| **涨跌停限制**      | 校验委托价格 + 拒绝超出范围订单                     | price >= limit_down && price <= limit_up |
| **停牌股票**       | 查询股票状态 + 拒单 + 提示                       | stock.status检查            |
| **涨跌停无法卖出**   | 状态判断 + 禁止反向订单                          | limit_up/limit_down判断     |
| **股票复牌**       | 状态更新 + 恢复交易 + 通知                        | 定时同步状态                 |
| **股票退市**       | 强制平仓提醒 + 持仓处理 + 限制 | status =交易                  DELISTED + 告警     |
| **送转股/分红除权** | 持仓更新 + 成本调整 + 盈亏重算                     | 自动调整算法                 |

### 3.5 并发安全类

| 场景               | 处理方案                                    | 技术手段                    |
|------------------|-------------------------------------------|---------------------------|
| **并发下单超扣**    | 行级锁 + 乐观锁 + 事务                       | SELECT ... FOR UPDATE      |
| **并发撤单**       | 状态校验 + CAS更新                           | version字段                |
| **并发修改持仓**    | 事务隔离 + 乐观锁                            | Repeatable Read + version  |
| **并发撮合同一订单** | 订单状态锁 + 分布式锁                         | Redis分布式锁               |
| **死锁**          | 统一加锁顺序 + 超时机制                       | 锁超时 + 死锁检测            |

### 3.6 风控类

| 场景               | 处理方案                                    | 技术手段                    |
|------------------|-------------------------------------------|---------------------------|
| **单笔超大订单**     | 金额限制检查 + 拆分提醒                       | max_order_amount限制       |
| **单日累计限额**     | 日累计统计 + 超限拒单                         | Redis日计数器               |
| **频繁撤单**        | 撤单限流 + 告警                              | 滑动窗口计数器               |
| **异常价格**        | 价格偏离度检查 + 预警                         | 均线偏离 > 5% 预警          |
| **洗钱检测**        | 大额快进快出监控 + 风控审查                    | 行为分析 + 人工审核          |
| **老鼠仓检测**      | 关联交易监控 + 关联账户限制                    | 图数据库关联分析              |

### 3.7 数据一致性类

| 场景               | 处理方案                                    | 技术手段                    |
|------------------|-------------------------------------------|---------------------------|
| **资金流水缺失**    | 对账检测 + 自动补录                          | 差异检测 + 补录脚本          |
| **成交记录丢失**    | Kafka重放 + 幂等恢复                         | 事件重放机制                |
| **持仓数据不一致**   | 对账修复 + 手动干预                          | 差异修复脚本                |
| **订单状态不一致**   | 状态机纠正 + 事务补偿                        | 状态机设计                  |
| **对账差异未处理**   | 定时扫描 + 自动修复 + 人工介入                | cron定时任务                |

## 七、监控与告警

### 7.1 监控指标

- 请求延迟（p50, p95, p99）
- 错误率
- 并发数
- 队列深度
- 资金差异数

### 7.2 告警规则

- 资金差异 > 0.01元：Warning
- 资金差异 > 1元：Critical
- 错误率 > 1%：Warning
- 错误率 > 5%：Critical
- 服务延迟 > 1s：Warning
